app Folder
Contains all needed data for the framework

models Folder
Contains all models for the framework(isnt as a mere folder since all classes will be autoloaded in index file)

views Folder
Contains all files which can be displayed in a controller

controller Folder
Contains all controller classes

libraries Folder
Contains PWEL itself and various other standalone classes

AutoInject.php
This is a class where all methods will be executed before controller startup

config.ini
The file where much optional configuratiosn can be done

html Folder
Could have any other possible name aslong .htaccess and index file is in it

.htaccess
Needed for correct routing

index.php
Contains configuration and startup from PWEL

System of PWEL


The Php Worker Environment Lite is based on MVC and is able to do HMVC and mutli project manamgent/project parting.

MVC(Model/View/Controller)
This system allows to code a structured site


HMVC(Hirachical Model/View/Controller)
Through the autosearch option the framework will browse through all directories in controller/views and use the destinated file


Multi project management/project parting
You can create as much projects you want or seperating it by creating a folder and put a configurated index file and the .htaccess in it.
You use a new namespace for every new part or project to seperate them from each other.

Classes of PWEL

All classes of PWEL are standalone classes and so you can call them everywhere.

Example: 

$pc = new PWEL_CONTROLLER();
$pr = new PWEL_ROUTING();
if($pc->displayExists($param.".html"))
    $pc->display($param.".html");
else
    $pr->displayError();

List of main classes

PWEL class
Starts the framework and got else nothing really interesting.

PWEL_CONTROLLER class
Contains various function for the controllers like

  • display($filename,(optional)$variables)
  • displayExists($filename)
  • validateCss($filename)
  • validateJS($filenameOrUrl)
  • validateLink($path)

PWEL_ROUTING class
Handles the routing process. You can get some useful informations from it like

  • autoSearch($path,$file)
  • displayController($class,(optional)$mode)
  • requestRelativePath()

PWEL_URL class
Handles parameter behaviours and redirects
