display()
Description:
* Display a file from view folder
* Variables will be stored in class itself and will be accessable as normal variables
* in controller class
* Example:
* function index() {
* $this->testVariable = "hello!";
* $this->display("outputfile");
* }
*
* Outputfile:
* <?php print $testVariable.' im a test!';
*
* @param mixed $vars
* @param string $filename
Example:
//Sets variables for function
$var["xd"] = "hallo";
//Sets variables which will be extracted in function itself
$this->hallo = "xd";
//Calling a display function with $var array
$this->display("test",$var); //if you give no extension it will use php
test.php: 
print $xd."<br />";
print $hallo."<br />";
Output: 
hallo
xd

else

displayed file or error
getRegister()
Description:
* Returns a registered object
*
* "*" in name is a wildcard to spare the time writing full class names
* Example:
* PWEL_PLUGIN_HTML_HELPER => *_HTML_HELPER, *_HTML, *_HELPER
*
* @param string $name
* @return object
Example: 
//Creates a new instance
$pc = new PWEL_CONTROLLER();
//Contains a instance of PWEL_PLUGIN_HTML_HELPER
$html_helper = $pc->getRegister("*_HTML");
validateCss()
Description:
* Returns a validated css link tag
* @var string $file
* @return string
Example: 
//Creates a new instance
$pc = new PWEL_CONTROLLER();
//Put a validated css tag into the variable
$cssTag = $pc->validateCss("css/file.css");
validateJS()
Description:
* Returns a validated script link tag
* @var string
* @return string
Example: 
//Creates a new instance
$pc = new PWEL_CONTROLLER();
//Put a validated js tag into the variable
//$file can be a url aswell
$jsTag = $pc->validateJs("js/file.js");
validateLink()
Description:
* Returns a validated link
* from PWEL_URL
*
* @var string $file
* @return string
This function just return the function from PWEL_URL.
So theres no need to create an instance of PWEL_URL in controllers anymore just for validating links.
